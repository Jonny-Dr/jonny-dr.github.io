# 项目优化计划

## 1. 抽离公共样式和脚本

### 1.1 创建公共样式文件
- 创建 `css/common.css` 文件，包含所有页面共享的样式
- 包含：CSS变量、基础样式、导航栏、页脚、主题切换按钮样式
- 每个页面通过 `<link rel="stylesheet" href="css/common.css">` 引入

### 1.2 创建公共脚本文件
- 创建 `js/theme.js` 文件，包含主题切换的逻辑
- 每个页面通过 `<script src="js/theme.js"></script>` 引入

### 1.3 页面特定样式
- 每个页面只保留页面特定的样式，减少代码重复
- 例如：首页的文章列表样式、归档页的年月分组样式等

## 2. 优化分页功能

### 2.1 增强分页脚本
- 优化 `generate-pagination.js` 脚本，使其支持多个页面的分页
- 配置不同页面的分页参数（每页显示数量等）
- 确保首页、项目页、日常页等需要分页的页面都能自动分页

### 2.2 分页配置
- 归档和关于我页面不需要分页
- 其他页面根据内容量自动生成分页

## 3. 确保GitHub Pages兼容性

### 3.1 资源路径优化
- 确保所有资源路径使用相对路径，避免绝对路径
- 确保图片、CSS、JS等资源能够正确加载

### 3.2 构建工具避免
- 使用纯HTML、CSS和JavaScript，避免使用需要构建的框架
- 确保项目可以直接在GitHub Pages上运行

### 3.3 GitHub Actions优化
- 优化 `.github/workflows/pagination.yml` workflow
- 确保自动部署和分页生成正常工作

## 4. 代码组织和优化

### 4.1 文件结构整理
- 整理文件结构，使其更加清晰
- 创建 `css/` 和 `js/` 目录存放样式和脚本文件

### 4.2 代码质量优化
- 优化HTML结构，提高语义化
- 优化CSS代码，提高可维护性
- 添加必要的注释，提高代码可读性

### 4.3 响应式设计优化
- 确保所有页面在不同设备上都能正常显示
- 优化移动设备的用户体验

## 5. 实现步骤

1. 创建 `css/` 和 `js/` 目录
2. 提取公共样式到 `css/common.css`
3. 提取主题切换逻辑到 `js/theme.js`
4. 修改所有HTML文件，引入公共样式和脚本
5. 优化 `generate-pagination.js` 脚本
6. 测试分页功能
7. 测试GitHub Pages部署
8. 优化代码质量和响应式设计

## 预期成果

- 统一的页面风格，所有页面使用相同的公共样式
- 减少代码重复，提高可维护性
- 自动分页功能，支持文章内容的增量
- 完全兼容GitHub Pages，无需复杂的构建工具
- 清晰的代码结构和良好的用户体验
